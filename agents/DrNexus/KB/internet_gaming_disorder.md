# Internet Gaming Disorder — Orchestrator Reference

## Rapid Framing for Task Decomposition

When a task involves Internet Gaming Disorder (IGD), route to specialists using this framing:

| Sub-question | Best specialist(s) |
|---|---|
| Diagnostic criteria, clinical picture, comorbidity | ClinicalPsych |
| Prevalence, instrument validity, sampling bias | EpiScope |
| Reward mechanisms, neuroimaging, executive function | NeuroCogs |
| CBT efficacy, family interventions, care pathways | CarePath |
| Citation chains (King, Kuss, Brand, Lemmens, Petry) | LitScout |

## Key Disambiguation Points

- **ICD-11 status:** "Gaming Disorder" (6C51) — formally recognised.
- **DSM-5 status:** "Internet Gaming Disorder" — Section III research category only.
- **Not the same as broad PIU.** IGD is gaming-specific; do not generalise findings to other
  internet activities without explicit support.
- **Not the same as heavy gaming.** Functional impairment is the threshold, not hours played.

## Scope Boundaries for This Swarm

When the research task concerns IGD:
1. ClinicalPsych covers clinical criteria and comorbidity.
2. EpiScope covers measurement and prevalence (instruct them to name instrument + cut-off in every
   prevalence claim).
3. NeuroCogs covers mechanisms (instruct them to flag which findings are theoretical vs. empirical).
4. CarePath covers interventions.
5. Journalist must include: ICD-11 vs DSM-5 status disambiguation; prevalence with instrument named;
   impairment requirement stated.

## Mandatory Reviewer Trigger

Any IGD output that does not explicitly state "ICD-11 recognised vs. DSM-5 Section III" status
must be rejected by Reviewer-2. This is a required_element in swarm_config.yml.
